Realm of Dragn'tor

Realm of Dragn'tor is a 3D tower defence game where the player is tasked with defending their fort from other armies by placing towers that attack enemies.

Realm of Dragn'tor was developed using Unity and C# by a team of students acting as an indie studio named Code Damage.

Visit Code Damage repository to view the game's source code.
Visit Itch.io to view the game and download the zip.

About

Features

Realm of Dragn'tor is a 3D tower defence game with the following key features that I was responsible for creating;

Procedurally Generated Terrain:

Using Perlin Noise, the terrain was generated in a grid-based system. Perlin Noise algorithm provided a height map for the terrain similar to games like Minecraft that use approximate height values to identify the type of terrain.

For example, with heights ranging from 1 to 10, a water level of 3 means anything below would have water themed tiles, and a mountain level of 7 means anything above would have mountain themed tiles.

Artificial Intelligence:

Enemy units and towers automatically track and interact with other entities depending on line of sight or distance to a target. Depending on the type of entity, the unit or tower would perform the relevant actions when their target satisfies prerequisites like being in range.

All unit movement is enabled by the use of Unity's NavMesh system. After identifying the tiles that permits movement, a navigation mesh is applied that allows agents (units) to move around the terrain. When towers and buildings are placed on the terrain, the navigation mesh allocates an obstacle to the are that the building was placed, which results in the agents avoiding the obstacles to reach their intended target.

Pixelated Style Voxel Models:

Using MagicaVoxel, I created a number of voxel models that were used in the game. The models were exported as .obj files and imported into Unity. The models were then converted into prefabs that could be placed in the game.

Custom Event Handling System:

Constructed an event system that incorporates Unity's event system allowing easy inclusion and operation. This enabled a unified approach to attach events to elements that required interaction with a separate system such as mouse clicks, enemy spawns, tower placements and collision detections.

Other key features made by the rest of the team members:

  • Camera System: Taking inspiration from games like Total War, where the player can move and rotate the camera around the map.
  • MiniMap: A miniature representation of the terrain and position of units in an image overlay.
  • Menu System: Presenting different game states including a main menu, options screen, the main game and a game over screen.
